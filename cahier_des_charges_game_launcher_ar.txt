1. عرض المشروع
──────────────────────────────────────────────────────────────────────────────

إنشاء تطبيق ويب منصة ألعاب يسمح للمطورين المستقلين بعرض ألعابهم.

أدوار المستخدمين : مسؤول | مطور | مستخدم عادي

الأهداف :
  • تجميع الألعاب المستقلة
  • الترويج ودعم المطورين العرب والمشاريع الثقافية
  • تطبيق نظام التحقق الذي يضمن احترام القواعد الثقافية والأخلاقية والدينية
  • التحقق من الألعاب قبل النشر
  • السماح بتنزيل بسيط وسهل
  • واجهة حديثة : React (الواجهة الأمامية) + Django (الخادم)


2. أنواع المستخدمين والصلاحيات
──────────────────────────────────────────────────────────────────────────────

┌─ المسؤول ────────────────────────────────────────────────────────────────────┐
│  • الموافقة على أو رفض الألعاب                                               │
│  • اختبار الألعاب                                                             │
│  • التحقق من توافق المحتوى وفقاً للقواعد المحددة                              │
│  • إدارة حسابات المطورين                                                      │
│  • إدارة الفئات والوسوم                                                       │
│  • حذف الألعاب والحسابات                                                      │
│  • الوصول إلى الإحصائيات                                                      │
└──────────────────────────────────────────────────────────────────────────────┘

┌─ المطور ──────────────────────────────────────────────────────────────────────┐
│  • إنشاء حساب مطور                                                            │
│  • إرسال لعبة (العنوان والوصف والصور والملف)                                 │
│  • الاطلاع على حالة اللعبة (قيد الانتظار | معتمدة | مرفوضة)                  │
│  • يمكنه تصحيح اللعبة المرفوضة                                                │
│  • عرض الألعاب المستقلة و/أو الثقافية العربية                                │
└──────────────────────────────────────────────────────────────────────────────┘

┌─ المستخدم العادي ────────────────────────────────────────────────────────────┐
│  • تنزيل الألعاب بدون حساب                                                   │
│  • البحث والتصفية                                                            │
│  • مع الحساب : مكتبة شخصية + سجل التنزيلات                                   │
└──────────────────────────────────────────────────────────────────────────────┘


3. المميزات والوظائف
──────────────────────────────────────────────────────────────────────────────

المميزات الرئيسية :
  • تصفح الألعاب
  • تنزيل اللعبة
  • البحث والتصفية المتقدمة
  • إرسال الألعاب من قبل المطورين
  • التحقق الإلزامي من قبل المسؤول
  • مكتبة المستخدم (الألعاب المنزلة)
  • تسليط الضوء على الألعاب التي أنشأها مطورون مستقلون وعرب

سير العمل للنشر :
  1. إرسال اللعبة من قبل المطور
  2. حالة "قيد الانتظار"
  3. الاختبار والتحقق من قبل المسؤول
  4. الموافقة أو الرفض
  5. النشر العام أو إخطار برفض


4. الهندسة المعمارية التقنية
──────────────────────────────────────────────────────────────────────────────

الخادم – DJANGO
────────────────
نماذج البيانات :
  • User (المستخدم)
  • Game (اللعبة)
  • Screenshot (لقطة الشاشة)
  • Category (الفئة)
  • Library (المكتبة)
  • DownloadHistory (سجل التنزيلات)

نقاط نهاية API :
  • POST   /api/users/login/                (المصادقة)
  • POST   /api/games/games/                (إنشاء لعبة)
  • GET    /api/games/games-list/           (قائمة الألعاب)
  • GET    /api/games/games-list/{id}/      (تفاصيل اللعبة)
  • PATCH  /api/games/games/{id}/           (الموافقة/رفض من المسؤول)

الواجهة الأمامية – REACT
─────────────────────────
الصفحات :
  • Home                                    (الصفحة الرئيسية)
  • Browse                                  (استعراض الألعاب)
  • Game Details                            (تفاصيل اللعبة)
  • Download                                (صفحة التنزيل)
  • Login / Register                        (تسجيل الدخول والتسجيل)
  • Dev Dashboard                           (لوحة تحكم المطور)
  • Admin Dashboard                         (لوحة تحكم المسؤول)

المكونات القابلة لإعادة الاستخدام :
  • GameCard                                (بطاقة اللعبة)
  • FilterPanel                             (لوحة التصفية)
  • ScreenshotGallery                       (معرض لقطات الشاشة)
  • UploadForm                              (نموذج الإرسال)
  • LibraryList                             (قائمة المكتبة)


5. القيود التقنية
──────────────────────────────────────────────────────────────────────────────

  • الواجهة الأمامية      : React
  • الخادم                : Django + Django REST Framework (DRF)
  • قاعدة البيانات       : قيد التحديد (PostgreSQL موصى به)
  • تخزين الملفات         : محلي أو سحابي (AWS S3، إلخ)
  • المصادقة              : JWT (JSON Web Tokens)
  • دعم متعدد اللغات     : العربية والإنجليزية
  • تصميم سريع الاستجابة : هاتف وجهاز لوحي وسطح المكتب


6. المتطلبات غير الوظيفية
──────────────────────────────────────────────────────────────────────────────

  • الأداء السريع والسلس
  • كود نظيف وموثق جيداً
  • أمان قوي (التحقق من البيانات والحماية من التهديدات)
  • توثيق تقني شامل
  • إمكانية الوصول (WCAG)


7. المسلمات والمخرجات
──────────────────────────────────────────────────────────────────────────────
